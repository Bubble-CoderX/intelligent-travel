## 二、各阶段实施详情

### 第1阶段：核心基础 + 定位（O23 + O1 + O2）

#### O23 项目定位升级
- 新建 `docs/O23-项目定位与差异化.md` 文档
- 聚焦"带娃/带老人家庭旅行者"定位
- 包含差异化对比表、核心价值一句话、30秒演示脚本

#### O1 偏好系统重构
**实现内容：**
- 新建 `profile_extractor.py`：从对话自动提取出行档案（人数/构成/年龄/风格/兴趣/饮食/住宿/过敏/预算）
- `intent_router.py`：每条用户消息自动调用 `extract_travel_profile()`
- `PreferencesDrawer.vue` 重写为三区布局：出行档案（只读+删除）/ 精确设置（可编辑）/ 系统信息
- 预算等级自动计算（≤200穷游/≤500经济/≤1000舒适/>1000奢华）
- 中文key映射（food→美食, economic→经济 等全显示中文）

**遇到的问题及解决：**
- 偏好面板显示英文（food/economic）→ 新增 interestMap/budgetTierMap 等映射表，所有值显示中文
- 问"我的过敏史"返回 None → 意图识别把查询当保存，value为空时走查询逻辑+中文key映射
- 天气城市名显示英文（Guangzhou）→ weather.py 新增45个城市英中映射 + intent_router.py 也翻译
- 删了偏好后对话又自动恢复 → 撤回了删除追踪功能（用户认为不需要）

### 第2阶段：O2 行程规划质量升级

**实现内容：**
- `trip_service.py` 接入 RAG 知识库检索（`retrieve_knowledge`）
- 自动调研：无知识库时自动调用 `auto_expand()` 生成
- Prompt 大升级：景点过滤/预算约束/健康信息/交通规则/餐饮联动
- 出发地处理：消息提取 > profile存储 > IP定位
- 大交通/当地交通推荐
- 人均预算计算：total = daily_budget × group_size × days

**遇到的问题及解决：**
- 知识库检索返回其他城市脏数据 → 改用 `spot_name` 精确匹配替代语义搜索
- LLM返回非法JSON导致行程生成失败 → schemas.py 所有字段改为可选 + 清洗逻辑（负数预算→0）
- UnboundLocalError: group_size 未定义 → 提前定义到函数前部
- DeepSeek flash模型JSON不稳定 → 切回默认模型（pro）

### 第5阶段：O8 + O9 + O26 + O27

#### O8 图片文件上传（千问VL）
- 新建 `qwen_vl_client.py`（通义千问VL多模态客户端）
- `chat.py` /chat/image 端点：图片识别 + 自动调研知识库
- 图片保存到 `data/uploads/` 目录，返回 URL
- 前端：data URL 临时显示 → 服务器 URL 持久化

**遇到的问题及解决：**
- DeepSeek API不支持 image_url → 换用千问VL-Max（qwen-vl-max）
- 图片识别不准确 → 调整 prompt 为"先描述再识别"的分步分析
- 图片刷新后消失 → 图片存到磁盘 + URL 存入 metadata
- AI回复刷新后丢失 → 传正确的 session_id 使消息保存到数据库
- 两条重复回复 → 去掉 emit('send')，直接手动添加消息

#### O9 流式输出
- `llm_client.py` 新增 `call_llm_stream()` 函数
- `chat.py` 新增 `/chat/stream` SSE 端点
- `chat.ts` 前端支持流式接收（逐字显示效果）
- TRIP_PLAN 保持非流式（JSON输出）

#### O26 Prompt注入防护
- `safety.py` 新增 18 条注入检测模式
- 检测到注入攻击 → 直接拦截，返回"请正常提问"

## 四、所有遇到的问题汇总

### 代码层面
| 问题 | 根因 | 解决方案 |
|------|------|---------|
| ValidationError反复出现 | schemas.py字段全必填 | 所有字段改为Field(default=xxx) |
| UnboundLocalError: group_size | 变量使用在定义前 | 提前到函数前部 |
| JSON解析失败 | LLM输出不规范 | 清洗逻辑 + 重试机制 |
| 偏好查询返回None | 意图识别把查询当保存 | value为空时走查询逻辑 |
| 图片识别失败 | DeepSeek不支持image_url | 换用千问VL-Max |
| PDF文件名中文乱码 | latin-1编码不支持 | RFC 5987 URL编码 |
| 时间显示不一致 | SQLite CURRENT_TIMESTAMP是UTC | 全部改为UTC+8本地时间 |
| 主动服务时间错误 | proactive.py用INSERT没指定created_at | 增加UTC+8参数 |
| 知识库检索脏数据 | 语义搜索返回其他城市 | 改用spot_name精确匹配 |
| 知识库文件截断 | max_tokens=4000太小 | 提升到10000 + 超时120秒 |
| 飞行票价不准 | 公式0.35元/km不准确 | 改为LLM根据知识估算二等座票价 |
| 闲聊回复太短 | prompt限制"1-3句" | 改为"3-5句" + max_tokens 300→500 |
| 图片刷新后消失 | imageData存内存不持久化 | 图片存磁盘 + URL存metadata |
| 两条重复回复 | emit('send')走聊天流程 + 图片分析 | 去掉emit，直接添加消息 |
| 清单不显示 | 前端找cat.name但数据用category | cat.name \|\| cat.category |
| 会话时间刷新后变化 | SQLite用UTC时间 | 全面改为UTC+8本地时间 |

### 设计决策
| 决策 | 原因 |
|------|------|
| 跳过O3(Coze) | 外部依赖风险高，非核心功能 |
| 跳过O10(轻量模型) | flash模型JSON不稳定 |
| 跳过O24.3(局部修改) | trip_plans/trip_history双表ID映射复杂 |
| 跳过O15(情绪感知) | P3优先级低，准确率有限 |
| 图片上传用千问VL而非DeepSeek | DeepSeek当前API不支持image_url |
